# 支付宝商户进件接口

## 重要原则 ⭐

## ant.merchant.expand.indirect.image.upload（图片上传）

**接口用途**：直付通商户进件的图片上传

**文档地址**：https://opendocs.alipay.com/solution/d0814d9c_ant.merchant.expand.indirect.image.upload

**请求对象**：AntMerchantExpandIndirectImageUploadRequest

**响应对象**：AntMerchantExpandIndirectImageUploadResponse

**调用方式**：文件上传接口（multipart/form-data格式）

**注意**：

- 本接口为文件上传类接口，与普通OpenAPI接口的入参构造格式不同
- 如果没有使用官方SDK发起OpenAPI调用，自行构造HTTP请求时，需要使用multipart/form-data格式发起请求，注意以下事项：
    1. 文件参数单独作为一个part，key为参数名，内容为文件，需要携带完整文件名（例如：Content-Disposition: form-data; name="
       file_content"; filename="example.png"），缺少文件名或后缀，会提示为类型错误
    2. 其余参数，对于v2版本的协议，公共参数需要使用query传入，业务参数使用query或body传入，使用body时每个字段作为一个单独part；v3协议需要按接口约定位置传入

### 公共请求参数

| 参数               | 类型     | 必填 | 说明                            |
|------------------|--------|----|-------------------------------|
| `app_id`         | string | 是  | 支付宝分配给开发者的应用ID                |
| `method`         | string | 是  | 接口名称                          |
| `format`         | string | 否  | 仅支持JSON                       |
| `charset`        | string | 是  | 请求编码格式，如utf-8                 |
| `sign_type`      | string | 是  | 签名算法类型：RSA2或RSA               |
| `sign`           | string | 是  | 商户请求参数的签名串                    |
| `timestamp`      | string | 是  | 发送请求的时间，格式yyyy-MM-dd HH:mm:ss |
| `version`        | string | 是  | 接口版本，固定为1.0                   |
| `app_auth_token` | string | 否  | 应用授权令牌                        |

### 业务请求参数

| 参数             | 类型     | 必填 | 说明                                   |
|----------------|--------|----|--------------------------------------|
| `image_type`   | string | 是  | 图片格式，支持：bmp、jpg、jpeg、png、gif（3-16字符） |
| `file_content` | byte[] | 是  | 图片二进制字节流，最大为10M（10485750字节）          |

### 公共响应参数

| 参数         | 类型     | 必填 | 说明      |
|------------|--------|----|---------|
| `code`     | string | 是  | 网关返回码   |
| `msg`      | string | 是  | 网关返回码描述 |
| `sub_code` | string | 否  | 业务返回码   |
| `sub_msg`  | string | 否  | 业务返回码描述 |
| `sign`     | string | 是  | 签名      |

### 业务响应参数

| 参数         | 类型     | 必填 | 说明                   |
|------------|--------|----|----------------------|
| `image_id` | string | 是  | 图片在文件存储平台的标识（256字符内） |

### 错误码

| 错误码                       | 说明                | 解决方案                               |
|---------------------------|-------------------|------------------------------------|
| SYSTEM_ERROR              | 系统繁忙              | 服务器异常，可能发生了网络或者系统异常，商户可以用同样的请求发起重试 |
| EXCEED_MAX_SIZE           | 超过单个图片允许的最大值（10M） | 请适当压缩图片大小以免超过上传限制                  |
| IMAGE_UNSAFE              | 文件未通过安全校验         | 图片中可能注入了恶意代码，请重新制作图片               |
| INVALID_IMAGE_FORMAT      | 文件格式不支持           | 请不要使用非常用的图片格式，如CMYK                |
| MERCHANT_ILLEGAL_ARGUMENT | 参数格式非法            | 参数格式非法，请检查                         |

### 请求示例

```bash
curl 'https://openapi.alipay.com/gateway.do?charset=UTF-8&method=ant.merchant.expand.indirect.image.upload&format=json&sign=${sign}&app_id=${appid}&version=1.0&sign_type=RSA2&timestamp=${now}' \
 -F 'image_type=jpg' \
 -F 'file_content=@/home/admin/test.jpg' 
```

### 响应示例

```json
{
  "ant_merchant_expand_indirect_image_upload_response": {
    "code": "10000",
    "msg": "Success",
    "image_id": "a5d373f6-3e79-405f-9993-fb7ea051c372.jpg"
  },
  "sign": "ERITJKEIJKJHKKKKKKKHJEREEEEEEEEEEE"
}
```

## ant.merchant.expand.bizaccess.order.check（业务开通准入预校验）

**接口用途**：用于服务商代商户业务开通准入预校验。本接口仅执行绝大部分"校验逻辑"，不会实际执行业务申请的提交。可提前识别并解决可能的问题，优化整体流程效率。

**文档地址**：https://opendocs.alipay.com/isv/42f27645_ant.merchant.expand.bizaccess.order.check

**请求对象**：AntMerchantExpandBizaccessOrderCheckRequest

**响应对象**：AntMerchantExpandBizaccessOrderCheckResponse

### 公共请求参数

| 参数             | 必填 | 说明                            |
|----------------|----|-------------------------------|
| app_id         | 是  | 支付宝分配给开发者的应用ID                |

| charset        | 是  | 请求编码格式，如utf-8                 |
| sign_type      | 是  | 签名算法类型：RSA2或RSA               |
| sign           | 是  | 商户请求参数的签名串                    |
| timestamp      | 是  | 发送请求的时间，格式yyyy-MM-dd HH:mm:ss |
| version        | 是  | 接口版本，固定为1.0                   |
| app_auth_token | 否  | 应用授权令牌                        |
| biz_content    | 是  | 请求参数的集合，所有业务参数必须放在此参数中传递      |

| `merchant_logon_id`        | string          | 可选   | 商户登录号（1-64字符）                                                                   |
| `optional_front_prod_code` | array           | 可选   | 可选开通的产品（1-20个）                                                                  |
| `legal_info`               | object          | 可选   | 法人证照                                                                            |
| `benefit_info`             | object          | 可选   | 受益人证照                                                                           |
| `need_register_merchant`   | boolean         | 可选   | 是否需要代入驻                                                                         |
| `merchant_type`            | string          | 可选   | 商户类型（0-64字符）                                                                    |
| `legal_person_logon_id`    | string          | 可选   | 法人登录号（0-256字符）                                                                  |
| `external_id`              | string          | 可选   | 外部单号（0-128字符）                                                                   |

#### isv_commission_info（服务商抽佣信息）子参数

| 参数                           | 类型      | 必填   | 说明                                                                                                                                                   |

| `long_term`                  | boolean | 是    | 是否是长期模式场景模式，为true时不用传抽佣时间                                                                                                                            |

| `business_scope`    | string | 可选 | 经营范围（0-2048位）                     |
| `reg_capital`       | string | 可选 | 注册地址（0-2048位）                     |
